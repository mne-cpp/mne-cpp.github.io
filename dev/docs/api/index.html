MNE-CPP API Reference

This section is the canonical reference for every public class in the MNE-CPP C++ libraries. It is generated automatically from the in-source Doxygen comments and a curated registry of public symbols, so the API view always matches the headers that ship with the build.

The registry is maintained at doc/api_registry.json and the generation pipeline is specified by TASK 18 in the v2.3.0 dev-notes.

Pages for individual classes are written to docs/api/<module>/*.mdx by tools/doxy2mdx/doxy2mdx.py and are excluded from version control. Run doc/build-api-docs.sh (or trigger the api-docs GitHub workflow) to populate them.

How is this page generated?​

The pipeline has three stages and is invoked end-to-end by doc/build-api-docs.sh:

  1. Doxygen parses the public headers in src/libraries/** using doc/Doxyfile and writes XML to doc/xml_out/xml/.
  2. tools/doxy2mdx/doxy2mdx.py translates the XML into MDX pages, cross-referencing the registry to drop unregistered internals and emitting a deterministic sidebars.api.generated.ts fragment.
  3. Docusaurus consumes the MDX plus the sidebar fragment to render the navigable /api/ section you are reading. Math is rendered through KaTeX (remark-math + rehype-katex) and code blocks are highlighted via Prism with cpp, cmake and bash support.

Adding a new public class​

  1. Add a new entry to doc/api_registry.json under classes. The minimum schema is:

    {
      "name": "MyClass",
      "module": "fiff",
      "header": "fiff/my_class.h",
      "origin": "custom",
      "documented": true
    }

  2. Make sure the header carries a Doxygen @brief and the SPDX header block introduced by TASK 17.

  3. Re-run doc/build-api-docs.sh — the validator will fail the build if the registered class cannot be resolved in the Doxygen XML, or if a public class is found in Doxygen output that is not in the registry.

For the full schema and contribution workflow see doc/api_registry.json and the TASK 18 specification.
